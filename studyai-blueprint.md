## 1. Project Overview

### Vision

StudyAI es una plataforma SaaS multiusuario con dos tipos de usuario: **estudiantes** y **maestros**.

Los **estudiantes** suben documentos académicos (PDF, .docx, .pptx) organizados en proyectos y generan recursos de estudio con IA: resúmenes, mapas conceptuales, exámenes simuladores y flashcards. Todo con Claude en streaming, en tiempo real.

Los **maestros** también suben documentos y generan exámenes, pero además pueden **publicarlos y compartirlos con sus estudiantes** a través de un código o link único. Los maestros ven los resultados y el progreso de sus alumnos en un dashboard de analíticas.

El modelo de negocio es freemium con Stripe Subscriptions. La plataforma está diseñada para escalar: los 4 recursos actuales son el punto de partida, y la arquitectura contempla agregar nuevos recursos y funcionalidades sin reestructurar el sistema.

### Goals

- Estudiantes: organizar material en proyectos y generar 4 tipos de recursos con IA
- Maestros: crear exámenes desde sus documentos y distribuirlos a sus alumnos
- Monetizar con suscripciones para cubrir hosting + API de Claude
- Arquitectura extensible: agregar nuevos recursos (líneas de tiempo, fichas de vocabulario, etc.) sin romper lo existente

### Success Metrics

- Estudiante completa el flujo completo en menos de 5 minutos
- Maestro publica su primer examen en menos de 10 minutos
- Tasa de conversión free → Pro mayor al 5%
- Costo de API Claude por usuario activo Pro menor al precio del plan

## 5. API Design

### Routes Overview

| Method | Path | Descripción | Auth |
|--------|------|-------------|------|
| GET | `/api/projects` | Lista proyectos del usuario | Sí |
| POST | `/api/projects` | Crea proyecto | Sí |
| GET | `/api/projects/[id]` | Detalle proyecto | Sí |
| PATCH | `/api/projects/[id]` | Actualiza proyecto | Sí |
| DELETE | `/api/projects/[id]` | Soft delete | Sí |
| POST | `/api/projects/[id]/documents` | Upload documento | Sí |
| DELETE | `/api/documents/[id]` | Elimina documento | Sí |
| GET | `/api/projects/[id]/resources` | Lista recursos | Sí |
| POST | `/api/projects/[id]/resources` | Genera recurso (SSE) | Sí |
| DELETE | `/api/resources/[id]` | Elimina recurso | Sí |
| POST | `/api/shared-exams` | Maestro publica examen | Sí (TEACHER) |
| GET | `/api/shared-exams/[shareCode]` | Info del examen (para estudiante) | No |
| POST | `/api/shared-exams/[shareCode]/attempts` | Estudiante envía respuestas | No* |
| GET | `/api/teacher/exams` | Maestro ve sus exámenes publicados | Sí (TEACHER) |
| GET | `/api/teacher/exams/[id]/results` | Maestro ve resultados | Sí (TEACHER) |
| POST | `/api/stripe/checkout` | Crea sesión checkout | Sí |
| POST | `/api/stripe/portal` | Portal de billing | Sí |
| POST | `/api/webhooks/stripe` | Webhook Stripe | No (firma) |

*El estudiante puede tomar el examen sin cuenta, pero si tiene sesión se guarda su student_id.

### Endpoints Críticos en Detalle

**POST `/api/projects/[projectId]/resources`** — Generación con SSE
```
Request: application/json
{
  type: "summary" | "concept_map" | "exam" | "flashcards",
  document_ids: string[],
  title: string
}

Response: text/event-stream (SSE)
  data: {"type":"start","resource_id":"..."}
  data: {"type":"chunk","content":"..."}
  data: {"type":"done","resource_id":"..."}
  data: {"type":"error","message":"..."}

Errores antes de abrir stream:
  403 — límite de generaciones del plan superado
  400 — document_ids vacío
  409 — documentos aún en PENDING/PROCESSING
```

**POST `/api/shared-exams`** — Maestro publica examen
```
Request: application/json
{
  resource_id: string,       // debe ser type=EXAM y del maestro autenticado
  title: string,
  instructions?: string,
  time_limit_minutes?: number
}

Response 201:
{
  id: string,
  share_code: string,        // "ABC12XYZ" — 8 chars aleatorios URL-safe
  share_url: string          // "https://studyai.com/exam/ABC12XYZ"
}

Errores:
  403 — usuario no tiene rol TEACHER
  404 — resource_id no existe o no pertenece al maestro
  400 — el recurso no es de tipo EXAM
```

**GET `/api/shared-exams/[shareCode]`** — Info del examen para el estudiante
```
Response 200:
{
  title: string,
  instructions?: string,
  time_limit_minutes?: number,
  total_questions: number,
  teacher_name: string,
  is_active: boolean
  // NO incluir las respuestas correctas ni explicaciones
}

Errores:
  404 — código no existe
  410 — examen inactivo (is_active: false)
```

**POST `/api/shared-exams/[shareCode]/attempts`** — Estudiante envía respuestas
```
Request: application/json
{
  student_name: string,
  student_email?: string,
  answers: [{ question_id: string, selected_index: number }],
  started_at: string         // ISO timestamp
}

Response 201:
{
  score: number,             // 0-100
  correct_answers: number,
  total_questions: number,
  results: [{                // Ahora sí se revelan las respuestas
    question: string,
    selected_index: number,
    correct_index: number,
    is_correct: boolean,
    explanation: string
  }]
}
```

**POST `/api/webhooks/stripe`** — Stripe webhook
```
Headers: stripe-signature (SIEMPRE verificar con constructEvent())

Eventos manejados:
  checkout.session.completed → subscription_status = PRO, guardar subscription_id
  customer.subscription.updated → actualizar estado y subscription_ends_at
  customer.subscription.deleted → subscription_status = CANCELED
```

- **SSE** con hook `useGenerationStream` para streaming de IA
- **Estado del examen** local en ExamPlayer (respuestas seleccionadas, timer)
- **No React Query en v1** — Server Actions + `revalidatePath` son suficientes

## 8. Authentication & Authorization

### Auth Flow

```
Landing → "Empezar gratis" → /auth/sign-up (email/password o Google)
  → Supabase crea auth.user
  → DB trigger crea registro en tabla users con role=STUDENT por defecto
  → Redirect a /onboarding (selección de rol: Estudiante o Maestro)
  → Actualizar users.role según elección
  → Redirect a /dashboard (layout según rol)
```

### Middleware — Role-Based Routing

```typescript
// middleware.ts
const STUDENT_ROUTES = ['/dashboard', '/projects', '/settings']
const TEACHER_ROUTES = ['/dashboard', '/projects', '/exams', '/settings']
const PUBLIC_ROUTES = ['/', '/pricing', '/auth/*', '/exam/*', '/onboarding']

// Después de verificar sesión, verificar rol:
// Usuario con role=STUDENT intentando ruta de maestro → redirect /dashboard
// Usuario con role=TEACHER intentando ruta de estudiante → redirect /dashboard
```

### Roles & Permissions

| Rol | Puede hacer |
|-----|-------------|
| STUDENT (Free) | 2 proyectos, 3 docs/proyecto, 15 generaciones/mes, tomar exámenes compartidos |
| STUDENT (Pro) | Proyectos ilimitados, docs ilimitados, 200 gen/mes |
| TEACHER (Free) | 2 proyectos, 3 docs, 15 gen/mes, publicar hasta 3 exámenes activos |
| TEACHER (Pro) | Proyectos ilimitados, exámenes ilimitados, 200 gen/mes, analíticas avanzadas |

**Row Level Security:** Las tablas `projects`, `documents`, `resources` y `shared_exams` tienen políticas RLS en Supabase que garantizan acceso solo a los datos del usuario autenticado. `exam_attempts` es accesible para el maestro dueño del examen.

### Session Management

Supabase Auth con cookies httpOnly. Middleware refresca el token en cada request. Cliente browser usa `createBrowserClient`, servidor usa `createServerClient` con cookies de Next.js.

## 17. Reglas No Negociables del Proyecto

1. **Zero secrets al cliente.** `ANTHROPIC_API_KEY`, `STRIPE_SECRET_KEY`, `SUPABASE_SERVICE_ROLE_KEY` nunca en código cliente ni en `NEXT_PUBLIC_*`.

2. **Ownership en todos los API routes.** Obtener `user_id` de la sesión (nunca del body/params). Verificar que el recurso solicitado pertenece al usuario antes de operar.
